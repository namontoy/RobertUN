# Week 1 — `alive` Heartbeat Node (ROS 2 Humble, Python)

This note explains how we created a minimal **heartbeat** ROS 2 node that publishes a Boolean message on the topic **`/alive`**.  
It’s meant as a reference for **future me** and for anyone who wants to reproduce the setup inside this repo.

---

## Why `alive` exists

In robotics, you want a quick, reliable signal that the system is running.

`alive` is a tiny node that:
- proves the **ROS 2 installation** is correct
- proves the **workspace + build system** works
- provides a “health signal” we can later use,
---

A ROS 2 workspace typically looks like:

<pre>

ros2_ws/
|-- src/      # Your packages live here
|-- build/    # Intermediate build files (generated by colcon)
|--install/  # Where your packages are installed (setup files live here)
|-- log/      # Logs for each colcon invocation

</pre>

**Important:**

If src/ is missing or empty, colcon build will show 0 packages.

## Create the workspace

<pre>
mkdir -p ~/RobertUN/jetson_ros2/ros2_ws/src
cd ~/RobertUN/jetson_ros2/ros2_ws
colcon build --symlink-install
</pre>

## Create the package 

Let's go to inside src:

<pre>
cd ~/RobertUN/jetson_ros2/ros2_ws/src

ros2 pkg create alive_py --build-type ament_python --dependencies rclpy std_msgs
</pre>

What this does:

- creates an ament_python package (Python-based ROS 2 package)

- declares dependencies:
  - rclpy = ROS 2 Python client library (Nodes, timers, publishers)
  - std_msgs = standard message types (like Bool)


**Create this file:** Check this repo for the file implementation

`ros2_ws/src/alive_py/alive_py/alive_node.py`

Make it executable using `chmod +x`

**Register the executable (so ros2 run works)**:

Edit: `ros2_ws/src/alive_py/setup.py` 

And Ensure the entry_points contains:

<pre>
entry_points={
    'console_scripts': [
        'alive_node = alive_py.alive_node:main',
    ],
},
</pre>

## Finishing...

**From the workspace root:**
<pre>
cd ~/RobertUN/jetson_ros2/ros2_ws
colcon build --symlink-install
source install/setup.bash
</pre>

## Run and verify

## Short command for `zsh` - for speed 

Because this machine uses `zsh`, the shell helper should use the `zsh` setup files, not the `bash` ones.

Save this function in:

`~/.zshrc`

Add:

<pre>
ros2ws_init() {
  source /opt/ros/humble/setup.zsh
  source ~/RobertUN/jetson_ros2/ros2_ws/install/setup.zsh
}
</pre>

Then reload your shell config:

<pre>
source ~/.zshrc
</pre>

After that, in any new terminal, you can do:

<pre>
ros2ws_init
</pre>

And then run ROS 2 commands without typing both `source` lines every time.

Example:

<pre>
ros2ws_init
ros2 run alive_py alive_node
</pre>

In another terminal:

<pre>
ros2ws_init
ros2 topic echo /alive
</pre>
